# これはなに

ども、もりたです。

以前、[年間700本ドキュメントを書く人間の技術ブログ執筆方法](https://zenn.dev/levtech/articles/9c33c2f6aca805)という記事をアップしたんですが、その記事末尾に技術ブログを書く上でのハードルについてのアンケートをつけておきました。回答数は少なかったですが、傾向としてパキッと面白いものが出たので、今回はその結果をご共有します。

# アンケートの結果

## アンケートの内容と意図

前提として、どんなアンケートをどんな意図で取ったのかについて整理します。
まず、アンケートの質問項目としては下記の通りでした。

```　text
# 質問項目
- 基本情報
    - 所属組織について
        - 選択：自社開発、SIer/SES、フリーランス、その他
    - 担当業務
        - 選択：BE, FE, フルスタック、ネイティブアプリ、汎用機、制御組み込み、インフラネットワーク、PM、PdM、コーポレート、データサイエンス、ITコンサル、QA、その他
    - 技術ブログの年間投稿数
        - 選択：0, 1, 2~5, 5~15, 16~50, 50~
- 技術ブログ執筆時のハードル
    - 記事執筆にポジティブか
        - 選択：とても同意する、少し同意する、あまり同意しない、まったく同意しない
    - 記事執筆にあたってのハードル
        - 複数選択：書くための環境がない、ネタが思いつかない、ネタはあるがまとまらない、文章がうまく書けない、書き終えられない、マサカリが怖い
    - 記事執筆に関して不安に思うこと
        - 自由記述
    - 記事を執筆して良かったこと
        - 自由記述
    - 「ドキュメント700本」記事への感想
        - 自由記述
```

アンケートの内容は前半と後半に分かれ、回答者の基本情報と技術ブログ執筆にあたってのハードルについての質問で構成されています。基本情報で得た属性と感じているハードルでクロス分析をしようという意図です。

## 結果の概要

アンケートの回答数は7件[^1]、全ての回答者に共通する課題は「ネタが思いつかない」「ネタはあるがまとまらない」であり、記事の年間執筆数が少ない群では「マサカリが怖い」が顕著でした。

また、回答件数が少なかったため、別の募集媒体（ランサーズ）でも同様のアンケートを募集しました。ランサーズでは16件の回答があり、投稿件数の多い少ないにかかわらず「ネタが思いつかない」「ネタはあるがまとまらない」が課題、記事の年間執筆数が少ない群では「文章がうまく書けない」が多かったです。

## 結果の考察

回答の件数が7件だったのでこれが統計的な事実だとはいえませんが、それでもかなりはっきり傾向が出たのは面白かったなと思います。

また、ランサーズでの回答では「マサカリが怖い」という回答が少なかったことから、Zennの回答者はサンプルとして偏りがありそうということもわかりました。実際、「Zennを利用しており」「技術ブログの書き方に興味があり」「技術ブログ執筆時のハードルアンケートに回答」しているというだけで、属性の偏りがあると言えそうです。

これらのことから何かストーリーを考えてみるならば、「日頃からZennなどで技術についての情報収集を行い、自身も技術ブログを書きたいと思っているが、自分が本来出したいアウトプットの量と実際に出せているアウトプットの量に差があると感じている」層に、「マサカリが怖い」と感じている人が多いということではないでしょうか。

# まとめとこれから

今回はアンケートをもとに、技術ブログを書くうえでどんなことが課題になっているのかについて考察しました。投稿件数に関わらず、「ネタが思いつかない」「ネタはあるがまとまらない」点が課題と思われているほか、年間投稿件数の少ない群では「マサカリが怖い」点が課題でした。

ただ、同時に疑問も湧いてきます。**なぜマサカリが怖いと思われているのでしょうか？**

マサカリが怖いという意見は漠然と共感できますが、マサカリという言葉は定義があやふやだったりして内実についてはあまり見えてこないのではないでしょうか？
