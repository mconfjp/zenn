# 記憶媒体とデータベースの歴史

## 1. 記憶媒体の歴史と発展

### 1.1 磁気記録媒体の誕生

コンピュータの補助記憶装置としての磁気記録媒体は、初期のコンピュータ開発と共に進化してきました。最初期の記録媒体としては紙のパンチカードが使われていましたが、1950年代から磁気記録媒体が本格的に利用されるようになりました。

### 1.2 磁気テープの発展

磁気テープは1920年代後半にドイツのフリッツ・フロイメルによって発明されましたが、コンピュータ用磁気テープとしては1951年に米国レミントンランド社（現Unisys）のUNIVACⅠに搭載されたものが最初の商用製品とされています[^1]。

磁気テープは初期のコンピュータにおいて主要なバックアップ媒体として利用され、現在でも大容量データの長期保存に使われています。1970年代には一般消費者向けにカセットテープが普及し、家庭用のデータ記録媒体として広く使われるようになりました。

磁気テープの特徴は以下の通りです：
- 大容量データを低コストで保存できる
- 長期保存に適している
- シーケンシャルアクセスのため、ランダムアクセスに比べて検索速度が遅い

現代でも磁気テープは大規模バックアップシステムとして使われており、2000年代に入ってからはLTO（Linear Tape Open）という統一規格が主流となっています。Googleやマイクロソフトなどの大手IT企業も現在でもデータセンターでのバックアップに磁気テープを使用しています[^2]。

### 1.3 磁気ディスクの発展

大容量の外部記憶装置として広く普及した磁気ディスク装置は、磁気ドラム装置の次に登場しました。複数の記録円板を重ねて回転させることで、体積あたりの容量を大幅に増加させることが可能になりました[^3]。

初期の主要な磁気ディスク技術の発展は以下の通りです：
- 1957年：IBMがRAMAC（容量5メガキャラクタ、直径24インチディスク、50枚）を実用化
- 1963年：IBMが磁気ディスクを交換できる磁気ディスクパック方式のIBM1311（2メガキャラクタ、14インチ、6枚）を開発
- 1964年：IBM2311（7.25メガバイト、14インチ、6枚）
- 1966年：IBM2314（29.2メガバイト、14インチ、11枚）
- 1971年：IBM3330（100メガバイト、14インチ、11枚）
- 1973年：IBM 3340（通称ウィンチェスタ）ディスクとヘッドの組合せを固定化

小型化の進展も見られました：
- 1979年：8インチディスク登場
- 1980年：5.25インチディスク登場
- 1987年：3.5インチディスク（富士通）
- 1990年：2.5インチディスク（東芝）

ハードディスクドライブ（HDD）は磁気ディスクの技術を応用した製品で、複数の磁気ディスクを固い金属製のケースで覆った記憶装置として発展しました。HDDはコンピュータの主要な記憶媒体となり、現在でも数TBの記憶が可能な主力記憶媒体となっています[^4]。

### 1.4 光学メディアとフラッシュメモリの登場

1980年代以降、新たな記憶媒体が登場しました：

1. **光学メディア**:
   - 1982年：コンパクトディスク（CD）がソニーとフィリップスにより商品化
   - 1990年代：DVD、その後Blu-rayディスクへと発展

2. **フラッシュメモリ**:
   - 1984年：東芝によるフラッシュメモリ開発
   - 2000年代：USBメモリやSDカードの普及
   - 2010年代：SSD（Solid State Drive）が主流化

特にSSDは、従来のHDDと比較して高速なデータアクセス、低消費電力、耐衝撃性という特徴を持ち、現代のコンピュータシステムに大きな変革をもたらしました。

## 2. データベースの歴史と発展

### 2.1 データベースの基本概念の誕生

データベースの概念は1959年にW.C.MaGee氏が発表した『Generalization: Key to successful electronic data processing』（概括：電子データ処理を成功させる秘訣）という論文で提示されました[^5]。

MaGee氏はこの中で「source file（源泉ファイル）」という概念を提唱し、データ重複の回避とシステム拡張の効率化のため、source fileにコンピュータ上の様々なデータを集約する考えを述べています。この時点ですでに「同じデータをあちこちに持っていると管理が大変である」という問題意識が存在していました。

「データベース」という言葉自体はアメリカ軍の軍事用語が起源で、「データ収集/利用基地」という意味で元々は「data base」と2語に分けていましたが、現在の「database」という表記になりました[^5]。

### 2.2 階層型・ネットワーク型データベース

最初の商用データベースは以下のように発展しました：

1. **ネットワーク型データベース（NDB）**:
   - 1963年：GE社のIDS（Integrated Data Store）は最初の商用データベースとして登場

2. **階層型データベース（HDB）**:
   - 1968年：IBM社のIMS（Information Management System）が登場

これらの初期データベースシステムは、データ間の関係を階層関係として表現し、データの位置をプログラムで特定してデータを取得するモデルでした[^6]。日本では1974年に富士通・日立・日本電気がネットワーク型データベースを発表し、日本の大企業でのデータベース普及が始まりました[^7]。

### 2.3 リレーショナルデータベースの革命

リレーショナルデータベース（RDB）は、1969年にIBMのエドガー・F・コッドが「Derivability, Redundancy, and Consistency of Relations Stored in Large Data Banks」という論文で最初に提唱しました。翌1970年には学術誌『Communications of ACM』に「A Relational Model of Data for Large Shared Data Banks」というタイトルで掲載され、関係モデルが広く知られるようになりました[^8]。

コッドのもたらした関係モデルの基礎的手法には以下があります：
- データベース操作の基礎に述語論理（predicate logic）を使用
- 関係代数（relational algebra）の定義
- 関係計算（relational calculas）の定義

リレーショナルデータベースの商用システムとしての発展は以下の通りです：
- 1971年：Software AGのADABAS（世界で最初に普及した商用RDBとされる）
- 1979年：Oracle Version 2（商用RDBMSの元祖と言われる）
- 1980年代前半：商業的にRDBMSが広く導入され始める
- 1982年：富士通 AIM/RDB 
- 1983年：IBM DB2
- 1984年：日立 RDB1

1980年代後半になると、コンピュータの性能向上によりRDBの実用性が高まり、1990年代以降はRDBが最も普及したデータベース形式となりました[^7]。

### 2.4 現代のデータベース技術

1990年代以降、データベース技術は多様化しました：

1. **オブジェクト指向データベース（OODB）**:
   - 1980年代半ば：複雑に構造化されたオブジェクト形式のデータを保存・共有するためのデータベース
   - 1997年：Oracle 8、富士通 Jasmine

2. **XMLデータベース（XMLDB）**:
   - 1998年：W3CによるXML標準仕様策定
   - 2001年：Microsoft SQL Server 2000、Oracle XML DB

3. **多次元データベース（MDDB）とデータウェアハウス**:
   - 1992年：インモンによるデータウェアハウスの定義
   - 1993年：コッドのOLAP提唱

4. **NoSQLデータベース**:
   - 1998年：「NoSQL」という言葉の誕生（カルロ・ストレッツィ）
   - 2000年代後半：分散処理やビッグデータの需要増加に伴い発展

5. **インメモリデータベース（IMDB）と列指向データベース（CODB）**:
   - 2005年：Oh-Pa 1/3、Oracle TimesTen In-Memory Database
   - 2010年：SAP HANA（インメモリデータベース）

現代のデータベース市場では、Oracle、IBM、Microsoftが主要ベンダーとなっています。クラウド環境の普及により、従来のプレイヤー以外にもSalesforceやSAPなどが参入し、データベース技術はより統合的なデータ管理プラットフォームへと進化しています[^7]。

## 3. 記憶媒体の進化がデータベース技術に与えた影響

### 3.1 アクセス速度の向上による処理能力の変化

記憶媒体のアクセス速度向上は、データベース技術の実用性に直接的な影響を与えました：

- **1970年代～1980年代前半**：
  当時の汎用コンピュータは、リレーショナルデータベースを使うには性能が不十分で、多くの企業がRDBのテスト段階であきらめていました。情報検索系での小規模利用や学習目的の使用が限界でした[^7]。

- **1980年代後半**：
  コンピュータの性能向上により、小規模ならばRDBが実用的になり始めました。この時期は「情報検索系にはRDB、基幹系にはNDB」という棲み分けが一般的でした[^7]。

- **1990年代～2000年代**：
  HDDの性能向上とコストパフォーマンスの改善により、RDBの基幹業務系への適用が可能になりました。RAID（Redundant Array of Inexpensive Disks）技術の登場（1987年提唱）も、データベースの信頼性と性能向上に貢献しました[^3]。

- **2010年代～現在**：
  SSDの普及により、データベースのI/O性能が飛躍的に向上しました。これによりインメモリデータベースの実用性が高まり、リアルタイム分析や高速トランザクション処理が可能になりました。

### 3.2 大容量化によるデータベース設計の変革

記憶媒体の大容量化は、データベースの設計思想にも影響を与えました：

- **1970年代～1980年代**：
  記憶媒体のコストが高く容量が限られていたため、データの正規化や重複排除が重視されました。これはリレーショナルデータベースの正規化理論とも合致し、RDBの普及を後押ししました。

- **1990年代～2000年代**：
  HDDの大容量化と低コスト化により、「データウェアハウス」という概念が実現可能になりました。大量の履歴データを保存し分析する需要が高まり、BIツールやOLAP技術が発展しました[^7]。

- **2010年代～現在**：
  ペタバイト級の大容量記憶が実現し、「ビッグデータ」時代が到来しました。従来のRDBでは処理しきれない大量データを扱うため、NoSQLデータベースや分散処理技術が発展しました。

### 3.3 新しいデータベースアーキテクチャの誕生

記憶媒体の多様化は、新しいデータベースアーキテクチャの誕生を促しました：

- **SSDの影響**：
  ランダムアクセスが高速なSSDの特性を活かすため、従来のB-treeインデックスとは異なるLSM-tree（Log-Structured Merge-tree）などの新しいデータ構造が普及しました。これはNoSQLデータベースの多くで採用されています。

- **インメモリ技術**：
  大容量メモリとSSDの普及により、データ全体をメモリに展開する「インメモリデータベース」が実用化されました。SAP HANAなどの製品は、トランザクション処理と分析処理を同時に高速化する「HTAP（Hybrid Transactional/Analytical Processing）」を実現しました[^7]。

- **分散ストレージアーキテクチャ**：
  大量のサーバとストレージを連携させる分散アーキテクチャが実現し、Hadoopエコシステムやクラウドネイティブなデータベースサービスが発展しました。

記憶媒体の進化は単にデータベースの性能向上をもたらしただけでなく、データの捉え方や活用方法そのものを変革し、データ中心のビジネス戦略を可能にしたといえます。
