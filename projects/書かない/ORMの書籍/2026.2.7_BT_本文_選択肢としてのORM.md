	- 先に、歴史的にどんな代替案があったのかをまとめましょう
- EmbeddedSQL
	- 概要
		- プログラミング言語の中にSQLを埋め込み、SQL部分のみプレコンパイルする
		- 例
			- EXEC SQL SELECT name INTO :name FROM users WHERE id = :id;
			- これを書いて変換するとなんか別のものになるとかだったような？？？　あと外部になんかファイルができて...みたいなのだったような...？？？
			- このコンパイルの流れ詳しく書く
	- 解決したこと
		- なし。この手法で始めてアプリケーションからSQLを呼び出した
- CLI（Call Level Interface）
	- 概要
		- 関数呼び出しでどうにかする
		- SQLPrepare(stmt, "SELECT name FROM users WHERE id = ?", ...);
	- 解決したこと
		- プリコンパイルをなくした
- ODBC
	- 概要
		- データベースごとに違う接続方法を抽象化している
		- その上で、SQLPrepare(stmt, "SELECT name FROM users WHERE id = ?", ...);みたいな書き方で問い合わせられる
	- 解決したこと
		- 接続手法やクエリの仕方とかを解決した
		- 言語の方言とかは吸収してないですよ
- リレーショナル指向プログラミング
	- 概要
		- Oの方をどうにかしようとした取り組み
	- 解決したこと
		- インピーダンスミスマッチの解消
		- けど、難解だし、デメリットも多くて流行らなかった
