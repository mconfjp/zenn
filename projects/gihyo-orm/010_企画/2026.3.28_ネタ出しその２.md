# 壁打ちの結果（2026.3.28）
## タイトル方向
- 「なぜORMは変なクエリをはくのか」
- 毎回「変なクエリ」の具体例から入って、ORMの仕組みを解剖する構造
- 読者は「変なクエリの理由」を知りに来たのに、気づいたらORMの全体像がわかってて、「面白いな」と思ってる、というのが理想

## 根っこにある欲求
- ORMって面白いなと思って欲しい
- 「何してるかわからん」が出発点で、「中身を知ったら面白かった」が着地
- アプリとDBの間にある課題を全て知りたい、が自分の動機

## 各回たたき台（5回）
1. なぜSELECT文に全カラムが並んでいるのか → ORMのマッピングの基本、そもそもORMとは何をしているのか
2. なぜ同じクエリが100回飛んでいるのか → N+1問題、レイジーローディングとイーガーローディング
3. なぜORMによってJOIN戦略が違うのか → ORM別のクエリ生成戦略、ActiveRecord系とDataMapper系の違い
4. なぜ型が合わないのか → DBの型とアプリの型のミスマッチ、TypedSQL等の現代的アプローチ
5. なぜORMは変なクエリをはくのか → 総括、インピーダンスミスマッチという根本問題、歴史と未来

## 課題
- 各回の「なぜ」に自分でまだ答えられない → 調査が先
- マイグレーションがこの構成に入ってない。別シリーズにするか、差し込むか要検討
- 技評の人からのフィードバック：ORMだけだと狭い、永続化レイヤーに繋がる必要がある → このタイトルならORMを入口に永続化レイヤー全体に触れられるはず

## 次にやること
- 各回の「なぜ」について調査する
- 調査した上で各回の構成を固める
- 企画書にまとめる
