# Prisma セットアップ手順

`code/prisma/` 配下で Prisma + TypeScript + MySQL 環境を構築する手順。

TypeScript系のORMは言語・ライブラリごとにフォルダを分け、それぞれに `package.json` を置く方針にしている（依存関係の分離のため）。

bunを使う。ts-nodeは不要（bunはTypeScriptをネイティブ実行できる）。

## 前提

- `code/docker-compose.yml` が存在している（MySQL 8.0、db: `orm_sample`、user/pass: `dev/dev`）
- bunがインストール済み（`bun --version` で確認）

---

## Step 1: package.json を作る

```bash
cd code/prisma
bun init -y
bun add @prisma/client
bun add -d prisma typescript @types/node
```

`package.json` の `scripts` に以下を追加しておく（`code/prisma/` からDockerを操作できるようにするため）：

```json
"scripts": {
  "db:up": "docker compose -f ../docker-compose.yml up -d",
  "db:down": "docker compose -f ../docker-compose.yml down",
  "migrate": "prisma migrate dev",
  "studio": "prisma studio"
}
```

---

## Step 2: TypeScript の設定

`bun init` で `tsconfig.json` が自動生成される。
Prisma との相性を考えて以下を確認・修正する：

- `"strict": true` — Prisma の型推論（nullable区別など）がフルに効く

---

## Step 3: Prisma の初期化

```bash
bunx prisma init
```

`prisma/schema.prisma` と `.env` が生成される。

`.env` を編集：

```
DATABASE_URL="mysql://dev:dev@localhost:3306/orm_sample"
```

`schema.prisma` の datasource を確認：

```prisma
datasource db {
  provider = "mysql"
  url      = env("DATABASE_URL")
}
```

---

## Step 4: スキーマを書く

最初はシンプルな User / Post から始める：

```prisma
model User {
  id    Int    @id @default(autoincrement())
  email String @unique
  name  String
  posts Post[]

  @@map("users")
}

model Post {
  id       Int    @id @default(autoincrement())
  title    String
  authorId Int    @map("author_id")
  author   User   @relation(fields: [authorId], references: [id])

  @@map("posts")
}
```

慣れてきたら `Comment`、`Tag`（多対多）を追加していく。

---

## Step 5: MySQL 起動 → マイグレーション

`code/prisma/` から実行できる：

```bash
bun run db:up                        # MySQL起動
bunx prisma migrate dev --name init  # マイグレーション
```

`prisma/migrations/` にSQLファイルが生成される。このSQLが記事で引用できる素材になる。

---

## Step 6: 動作確認

```bash
bunx prisma studio   # ブラウザでGUI確認
```

スクリプトを動かしたいとき：

```bash
bun run src/sandbox.ts
```

発行SQLを確認したいとき：

```bash
DEBUG="prisma:query" bun run src/sandbox.ts
```
