## 「世界からORMを消す」とは？
- さて、これからORMを消してみせたいわけですが、その前にみなさんORMがどんなものだかご存知でしょうか？
- 消すには対象を知らないといけません。屏風から出してみましょう。
### ORMとは何か？
- ORMはObject Relational Mapperの略
- 歴史的にはインピーダンスミスマッチというオブジェクト指向プログラミング言語とRDBの考え方の違いを吸収する層として生まれています。とはいえ普段利用者が見る顔としては、クエリの発行と、その結果の受け取りがメインのタスク。今回はそこに着目して、改めてORMが何をしているのか振り返ってみましょう。
- 利用するORMはPrismaです。
### どうやってクエリを発行するか？
- Prismaが発行する
- こうやってやる

```TypeScript

async function main() {
	const users = await prisma.user.findMany();
	console.log(users);
}
```

- どうなってるか？
	- prisma clientのuserモデルに全て書かれていて、そのuserモデルはスキーマ定義からの自動生成
	- Prismaは特殊。Rustでできたクエリエンジンが最適化した上でクエリ発行している
### どうやって結果を受け取る？
- 受け取ったクエリをオブジェクトに詰め替えるところをやる
- こうやってる
	- 実はさっきのコードと同じ

```TypeScript

async function main() {
	const users = await prisma.user.findMany();
	console.log(users);
}
```

- どうなってるか？
	- 受け取るオブジェクトをPrismaが自動生成してくれている
	- SQLの結果はスカラー値、行、列、行列の４種類しかない。列の特定はSelect句を見たらわかるし、行数はクエリの仕方である程度特定できる。結果のオブジェクトの形を特定することは難しいことではない。

## 消す
では消してみましょう
### クエリの発行
- まずはクエリの発行から。
	- TSの場合、こういうクエリの発行基盤がある。
	- pure TSでやってみる
- こうやって発行する
```TypeScript
const [rows] = await conn.query<mysql.RowDataPacket[]>("SELECT * FROM users");
```


### クエリの受け取り
- 受け取りはこうやる
- オブジェクトを作ってあげる
```TypeScript
type User = {
    id: number;
    email: string;
    name: string;
};
const users = rows as User[];
```
