# 気持ち
- 二ヶ月くらいこのことを調べて、IT産業の歴史はだいたい頭に入ったなあと思う
	- 1980年に機械式計算機がハーマン・ホレリスによってつくられ
	- 1951年に初めてプログラム可能な電子コンピュータのUNIVACが商用利用され
	- SAGE防空システムプロジェクトの中でソフトウェア産業というものができ
	- 1959年にData Bankという概念が生まれ
	- 1960年代はソフトウェア危機の時代で
	- 1967年にsimula67という最初のOOP言語ができ
	- 1969年にコッドがRDBの発端になる論文を出し
	- 1972年にSmalltalk-72が出てここでアラン・ケイがOOPという言葉を生んで
	- 1979年にOracleからOracle2が出て
	- 1980年代を通してOOPは研究されるが商用利用まではいかず、
	- 1983年にIBMのDB2が出て, 85年から一般公開しメインフレーム領域で伸長
	- 1990年からIBMがCOBOLからSmalltalkへの置き換えPRJを発足させ
	- 1991にXerox社がStarという初めてのGUIコンピュータを発売し
	- 1994年にTopLink for Smalltalkが世界で最初にORMとして流行り
	- 1995年にJavaが生まれ
	- 1996年にTopLink for Javaが出て
	- 1995年にWin95が出て
	- このGUIの元にOOPは新規領域で爆発的に伸びていく

## 構成
- これはなに
	- ORMの歴史をまとめる
		- みなさんORMを使うと思うけど、ORMがどのような課題意識を的にして出てきたのかを俯瞰的に学んだことはないんじゃないでしょうか
		- 今回の記事では米国のコンピュータ産業史を見ていくことで、どんな時代背景の中でORMが登場し、そして発展してきたのかをまとめます
	- 構成と意義
		- 構成
			- ORMの概要
			- 米国産業史
				- 1980s: 機械式計算機
				- 1950s: ソフトウェアの登場
				- 1960s: ソフトウェア危機の時代
				- 1970s: GUI, OOP, OS
				- 番外編: ソフトウェア産業とRDB
				- 1980s~1990s: GUIコンピュータの登場とOOPの商用利用
			- ORMの歴史
			- これからのORM
		- 意義
			- 歴史の俯瞰
			- 歴史を学ぶのってだるい、けど面白い、物語の上に私たちが生きており、どんな前提の上にいるのかわかる
- 歴史
	- ORMの概要
		- ORMとはOOPとRDBのマッピングをするもの
		- コード上で現れるオブジェクトと永続化レイヤーで保存されるリレーションは別の形式で保存されている。この変換をするのがORM
	- ORM登場までの
		- 1880s: 機械式計算機
		- 1950s: ソフトウェアの登場
			- プログラミング可能な電子コンピュータ
			- アセンブラとパンドラの箱
			- SAGE防空システムとソフトウェア・コントラクター
		- 1960s: ソフトウェア危機の時代
			- コーディングバイブルの歴史
			- 同時に60年代はタイムシェアリングが始まっている
				- [[OSという概念はタイムシェアリングシステムによるマルチユーザー概念から生まれている]]
				- OSの概念が生まれる
		- 1970s: GUI, OOP, OS
			- [[UNIXは1969年に開発され、1973年にC言語で書き直され移植性が高くなったことで普及した]]
			- [[GUIでOOPが発展するのは必然。GUIの元になった構想はOOPという言葉を生み出したアラン・ケイのダイナブックに（も）ある]]
			- 1969年にXerox（複写機メーカー）が未来のオフィス用テクノロジーを開発する目的でパロアルトリサーチセンターを設立する
			- 1973年にAltoの開発が始まる
				- これは1981年にStarとして一般向けに発売されるが、商業的には失敗した（クソ高かった）
		- 番外編: ソフトウェア産業とRDB
			- 1959年のデータバンク
			- 1969年にコッド博士の論文
			- 1970年代を通してのIBMの研究開発
			- 1979年にOracleのOracle2が初めての商用向けRDBとして登場
			- 1983年にIBMのDB2が登場し、COBOLと一緒に使われる
				- データベース抽象化ではないもの
					- ORMと比較可能なデータベース関連製品
			- なんかあのミニコンピュータ向けアプリケーション市場でのRDBがあったはず
		- 1980s~1990s: GUIコンピュータの登場とOOPの商用利用
			- 1981年Xerox Star、1984 Mac
				- こいつらもSmalltalkのGUIの影響を受けている
				- 一旦Smalltalkから離れて出てきたのが面白い
			- 1990年代初頭のIBMのSmalltalk（VisualAge）移行PRJ
				- GUIの移行需要が高まっていたし、社内ではClassCという内部開発言語でOOPをやっていた
				- 単なる採用言語の変更ではなく、開発環境の移行だった
			- Win95
				- GUIが一般に広がる
			- 1995年にJavaも登場
		- ORMの登場
			- 1994年に初めてのORM、そして2002年に初めてのOSS製ORMが登場する
	- ORMの歴史
		- 1994年にTopLink for Smalltalk
			- データマッパー型
		- 1995年にMySQL1.0がリリースされる、この時代はまだOODBMSへの期待も大きかった
		- 1996年にTopLink for Java
		- 2002年にTopLinkがOracle社に買収される
		- 2002年に初のJava向けOSS ORMであるHibernateが登場
		- 2005年にPHP向けのORMが登場
		- 2006年にRuby on RailsにActiveRecordが組み込まれる
		- 2007年にはWebフレームワークにはORMが標準装備されるように
	- これからのORM
		- 近年のやつ
		- ORMというのは本当か？　これはOじゃなくて、事業の変化ではないか
			- DDDとリポジトリ層
