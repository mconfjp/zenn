# 書籍執筆タスク定義

技術書典などのイベント向けに書籍を執筆するときのタスク定義。
各タスクの「準備完了条件」「やること」「完了条件」「サブタスク」を定義する。

---

## 全体フェーズ（プロジェクトで1回）

### フェーズ1: 企画構成

**準備完了条件：** 特になし（起点）

**やること：** 書籍全体の企画・章立て・セクション構成を決める。AIと壁打ちして方向性を固める。

**完了条件：**
- [ ] 章・セクション構成が決まっている
- [ ] 各セクションのコンセプトが言語化されている
- [ ] PJMが作成されている

**サブタスク：**
- [ ] AIと壁打ち（企画）
- [ ] 企画ファイル作成（`YYYY.M.D_企画_[説明].md`）
- [ ] 人間がレビュー・修正
- [ ] PJM作成（`YYYY.M.D_PJM.md`）

---

## セクションごとに繰り返すフェーズ

### フェーズ2: 調査対象一覧化

**準備完了条件：** フェーズ1が完了している（セクションのコンセプトが決まっている）

**やること：** `/permanent-harvest` スキルを使って、過去の調査資料からpermanentノート候補を抽出・一覧化する。

**完了条件：**
- [ ] このセクションで扱うべきトピックがすべて候補一覧に載っている
- [ ] 各候補に「コアアイデア」「ネタ」「要調査」が書かれている

**サブタスク：**
- [ ] 既存調査資料の洗い出し（`zettel/literatures/`, `zettel/permanents/` を確認）
- [ ] `/permanent-harvest` で候補一覧作成（`YYYY.M.D_permanent候補一覧.md`）
- [ ] 人間が候補一覧をレビュー・追記

---

### フェーズ3: 壁打ち → permanentノート化

**準備完了条件：** フェーズ2が完了している（候補一覧ができている）

**やること：** `/kabeuchi` スキルを使って候補を1件ずつ壁打ちし、理解を深める。理解できたものをpermanentノート化する。

**完了条件：**
- [ ] 候補一覧の全トピックについて壁打ちが終わっている
- [ ] 壁打ちで理解できたものがpermanentノート化されている

**サブタスク：**
- [ ] 各候補のステータスを候補一覧に追記しながら壁打ち
- [ ] 壁打ちで要調査が出たら調査を実施
- [ ] 理解できたものをpermanentノート化（人間が行う）

---

### フェーズ4: 箇条書き

**準備完了条件：** フェーズ3が完了している（主要トピックの壁打ちが終わっている）

**やること：** 人間が箇条書きで原稿の骨格を書く。

**完了条件：**
- [ ] セクションの主要な論点・事実が箇条書きになっている
- [ ] 箇条書きファイルが作成されている（`YYYY.M.D_箇条書き_[章名]_[セクション].md`）

**サブタスク：**
- [ ] permanentノートと候補一覧を参照しながら箇条書き作成

---

### フェーズ5: 清書・仕上げ

**準備完了条件：** フェーズ4が完了している（箇条書きがある）

**やること：** Claudeが箇条書きをもとに文章を清書する。

**完了条件：**
- [ ] 通読できる文章になっている
- [ ] 目標文字数の範囲内に収まっている

**サブタスク：**
- [ ] Claudeが箇条書きを清書
- [ ] 人間がレビュー・修正

---

## 最後のフェーズ（プロジェクトで1回）

### フェーズ6: 校正・組版

**準備完了条件：** 全セクションのフェーズ5が完了している

**やること：** 通読して校正し、ReViewなどで組版する。

**完了条件：**
- [ ] `/ai-check` による校正が完了している
- [ ] 印刷して朱入れ済み
- [ ] ReView形式に変換されている

**サブタスク：**
- [ ] `/ai-check` で全章校正
- [ ] 印刷して朱入れ
- [ ] ReView形式変換
- [ ] 最終確認

---

## 注記

- このREADMEは「型」の定義。実際の進捗はPJMで管理する。
- フェーズの定義・完了条件は運用しながら徐々に精度を上げていく。
- permanentノート化は必ず人間が行う（Claudeが勝手に書かない）。
